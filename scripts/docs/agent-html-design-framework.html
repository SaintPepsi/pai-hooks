03 — Hero

Document Header

Every document opens with a hero section. It contains a status badge, the document title with gradient text, a subtitle, and metadata.

Structure: .hero > .container > .hero-badge + h1 + p + .hero-meta. Optional: .hero-orb elements for ambient glow effects behind the hero.
🎯

Hero Rules

1 Badge states the document type: "Design Doc", "Response", "Analysis", etc.
2 Title uses gradient text via -webkit-background-clip: text
3 Subtitle is 1-2 sentences, max 560px width
4 Meta row shows date, version/status, author(s)
5 Orb glows are decorative only. Use max 2-3.

05 — Cards

Content Cards

The primary building block. Each card has a colored top-border gradient, an icon+title header, an optional badge, and body content. Apply a color class to the card element.

✅

Green Card — Success / Accepted

Accepted

Use for accepted items, completions, or positive outcomes. The green top-border and icon background signal success.

🔧

Orange Card — Warning / Refinement

Refined

Use for items that need adjustment, refinements, or warnings. The orange signals attention needed.

➕

Blue Card — Information / Addition

New

Use for new information, additions, or informational content.

❓

Accent Card — Primary / Question

Question

The default/primary card color. Also used for open questions or discussion items.

🍁

Maple Card — Agent-Specific

Maple

Use for content attributed to a specific agent. Replace the maple color with your agent's color.

⚙

Cyan Card — Technical / System

System

Use for technical details, system components, or infrastructure content.

06 — Insight Callouts

Callout Boxes

Highlighted boxes for key insights, important notes, or critical warnings. They stand out from the card flow.

💡 Default Insight (accent)

Use for key takeaways, important realizations, or strategic points. The gradient background and colored border draw attention.

🍁 Agent Insight (maple variant)

Use for agent-attributed insights. Swap the color class for your agent's identity color.

✅ Success Insight (green variant)

Use for positive outcomes, completed milestones, or summary callouts at the end of a section.

⚠ Warning Insight (orange variant)

Use for critical warnings, constraints, or things the reader must be aware of before proceeding.

15 — Options / Decisions

Presenting Choices

When presenting tradeoffs or decision points, use option boxes with pros/cons and a recommendation.

Option A: Standalone Repo Recommended

Shared repo that both sides clone and run independently.

+ Version control, CI, clean install story

+ Shared development workflow

- Requires repo setup coordination

Option B: Embedded in ~/.claude/

Lives inside each user's Claude config directory.

+ Zero setup, always available

- No shared version control

- Harder to keep in sync

Quick Reference

The Rules

Essential rules for agents generating HTML documents with this framework.

🎨 Use Tokens

Never hardcode colors. Always reference CSS custom properties.

📐 Start with Hero

Every document opens with a hero section. No exceptions.

📦 Cards for Content

Wrap meaningful content in cards. Color-code by semantic intent.

🔠 Two Fonts Only

Inter for prose. JetBrains Mono for code. Nothing else.

📏 880px Max Width

Container is 880px max. Comfortable reading width on all screens.

🌌 Dark Theme Only

The dark palette is the identity. No light mode variant.

🍁 How to Use This Framework

Copy the entire <style> block from this document into any agent-generated HTML. Then compose your content using the component patterns shown above. Every component is self-contained and composable. Mix cards, code blocks, tables, flow steps, and callouts as needed to tell your story clearly.

This document itself is generated using the framework it describes. What you see is what you get.

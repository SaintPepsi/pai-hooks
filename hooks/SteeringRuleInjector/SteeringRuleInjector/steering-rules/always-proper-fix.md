Always Proper Fix. Never present "quick fix" vs "proper fix" as options — always go with the proper fix. If you're tempted to offer a shortcut alongside the correct approach, skip the shortcut entirely. The right way is the only way. This applies to code fixes, architecture decisions, tool integration, and any situation where a hacky workaround exists alongside a correct solution.
